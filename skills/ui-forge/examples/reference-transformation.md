# Reference Transformation Example

Use this example when the user provides a screenshot, names a design direction, or asks for output similar to another skill/site.

## Weak default

Common weak result:

- copy the reference's navigation shape, hero geometry, capsule labels, font pairing, and accent placement as one bundle;
- change only colors and words;
- keep the reference's product assumptions even when the target product is different;
- avoid the reference too much and fall back to a generic gradient/card template.

## Better process

Extract traits, not the shell:

```text
Reference traits to retain:
- high type contrast;
- visible proof near the hero;
- tight metadata row;
- restrained accent color;
- authored section rhythm.

Target-derived choices:
- install command is the primary proof because this is a skill repo;
- before/after UI comparison demonstrates the skill better than a testimonial;
- docs/catalog routes need quieter reading surfaces than the front door.
```

Change at least two of:

- shell silhouette;
- navigation geometry;
- hero structure;
- accent placement;
- section rhythm;
- content proof;
- media treatment.

## Better output traits

- The page resembles the reference in principles, not in silhouette.
- Product-specific proof is more prominent than borrowed style.
- The target's language replaces generic marketing voice.
- The result can be recognized as its own product at thumbnail size.
