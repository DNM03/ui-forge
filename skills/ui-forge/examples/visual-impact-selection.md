# Visual Impact Selection Example

Use this example to learn how category resonance and visual magnetism affect direction selection. Do not reuse its palette, subject, typography, hero, or section sequence as a template.

## Brief

Create a fictional public landing page for a system that coordinates AI model releases across product, platform, and security teams. No customer proof, product screenshots, or established brand assets are supplied. Illustrative image generation is available.

## Three candidate routes

### Route A: luminous release topology

- **Category vocabulary:** routing paths, controlled luminosity, operational states, precise dark surfaces.
- **Primary carrier:** an illustrative topology object whose paths react to release-policy selection.
- **First-ten-second hook:** the object assembles from separate model, policy, and region layers into one governed route.
- **Page silhouette:** large product stage, concise workflow sequence, one dark evidence band, focused conversion close.
- **Truth strategy:** the topology is clearly illustrative; workflow states use representative labels rather than performance claims.
- **Motion system:** route assembly as the signature, slow energy travel as ambience, pointer inspection as the responsive layer.

### Route B: warm editorial operations journal

- **Category vocabulary:** restrained typography, paper-like surfaces, process writing, quiet diagrams.
- **Primary carrier:** editorial language and a static architectural render.
- **First-ten-second hook:** a calm headline beside the render.
- **Page silhouette:** heading-and-copy chapters followed by rounded information panels.
- **Risk:** it is elegant but could advertise consulting, architecture, or a design studio after changing the nouns. The render does not explain model orchestration.

### Route C: maximal command console

- **Category vocabulary:** dense grid, terminal chrome, neon state colors, mono labels, hard frames.
- **Primary carrier:** a fictional command dashboard.
- **First-ten-second hook:** multiple live panels and animated traces.
- **Risk:** strong category recognition, but the invented dashboard can masquerade as product evidence and the competing effects weaken hierarchy.

## Select and transform

Choose Route A. It wins category resonance and visual magnetism without inventing customers, metrics, or a fake production interface. Keep its technical vocabulary because it explains the mechanism, then transform it through one specific idea: policy paths physically converge into a release route.

Do not choose Route B merely because it is restrained and easy to defend. Restraint is not a substitute for product fit. Do not choose Route C merely because it is energetic. Energy without hierarchy or truthful evidence is not premium.

## Craft contract

```text
Two seconds: one luminous topology object dominates the hero
Five seconds: model release orchestration and the primary action are clear
Ten seconds: route assembly shows how policy changes the rollout path
Primary motion: topology assembly
Support motion: slow path travel | pointer inspection
Honest proof: representative workflow and inspectable illustrative output
Disclosure: one quiet footer note; no concept language in the hero
Scene change: a dark operational sequence following the luminous product stage
Delivery checks: short-height hero fit | mobile recompose | sticky-header clearance |
                 reduced motion | no fabricated customers, metrics, or testimonial
```
